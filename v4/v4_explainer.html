1. Phase2D Representation & Operations

The core idea of v4 is representing complex numbers as 2D real vectors (real, imag). This avoids trigonometric functions in the hot path, relying purely on GEMM/matmul operations.

Cayley Transform Rotation

Instead of learning angles (which require sin/cos), the model learns a skew-symmetric parameter a and computes rotation via the Cayley transform:

  • cos = (1 - a²) / (1 + a²)
  • sin = 2a / (1 + a²)

This guarantees an orthogonal matrix (pure rotation) without any trig.

PyTorch Implementation

def get_rotation_components(self) -> Tuple[torch.Tensor, torch.Tensor]:
    # For Cayley: cos = (1 - a^2) / (1 + a^2), sin = 2a / (1 + a^2)
    a = self.skew_params
    a_sq = a * a
    denom = 1.0 + a_sq
    
    cos_like = (1.0 - a_sq) / denom
    sin_like = (2.0 * a) / denom
    
    return cos_like, sin_like

3. Full Architecture Data Flow

The forward pass processes tokens through Phase2D embeddings, parallel banks, interference coupling, and an oscillatory backbone.

Model Forward Pass (v4/model.py)

# 1. Embed tokens to Phase2D
x = self.embed(input_ids=input_ids, mode='bpe')

# 2. Process through each bank in parallel
bank_outputs = {}
for bank_name, bank in self.banks.items():
    bank_outputs[bank_name] = bank(x, context=context)

# 3. Couple banks via interference
coupled = self.coupler(bank_outputs, context=context)

# 4. Process through backbone (Oscillatory SSM)
backbone_out, new_state = self.backbone(coupled, state=backbone_state)

# 5. Query global memory
memory_result = self.memory.read(backbone_out)
backbone_out = backbone_out + memory_result.values * self.memory_scale

# 6. LM head (Phase2D -> Real -> Logits)
lm_out = self.lm_head(backbone_out)
lm_real = phase2d_to_real(lm_out, mode='concat')
logits = self.output_proj(lm_real)

4. Oscillatory SSM (Backbone Dynamics)

The backbone uses a linear-time state-space model with oscillatory dynamics. The state evolves via rotation, damping, and input gating. Note that while the animation shows a single oscillator, the actual model maintains a bank of oscillators (e.g., dim = 256), each with its own learned frequency (skew_params) and decay rate (damping_param).

State Update Equation

h_{t+1} = gate · B x_t + (1 - gate) · damping · R(θ) h_t

  • Cyan vector: Current state h_t
  • Magenta vector: New input B x_t
  • Yellow arc: Rotation R(θ)
  • Dashed line: Damping

PyTorch Implementation

# Rotate previous state
h_rotated = self.rotation(h)

# Apply damping
damping = self.damping.unsqueeze(0).unsqueeze(-1)
h_damped = h_rotated * damping

# State update with gating
h = gate * Bx + (1 - gate) * h_damped

# Output projection
y_t = self.C(h) + self.D(x_t)

5. Backpropagation & Fine-Tuning

Gradients flow backward from the Cross-Entropy loss, through the Cayley transform and Phase2D layers, updating skew_params and damping_param.

Gradient Flow Details

Because the Cayley transform uses only basic arithmetic operations (multiplication, addition, division), PyTorch's autograd engine can automatically compute gradients for the skew_params without needing custom CUDA kernels for trigonometric derivatives.

# Complex matrix multiplication in Phase2DLinear
# (W_r + i*W_i) @ (x_r + i*x_i)
out_real = F.linear(x_real, self.weight_real) - F.linear(x_imag, self.weight_imag)

What is the Model Actually Learning?

As the gradients flow backward, they update specific parameters that control the physical properties of the phase-field simulation:

damping_param [dim] OscillatorySSM Learns the decay rate (memory retention). Values near 1.0 mean long memory; values near 0.5 mean rapid forgetting.

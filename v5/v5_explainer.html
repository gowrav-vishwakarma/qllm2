V5: Algebraic Language Model

A beginner-friendly walkthrough from text to next-token prediction

This explainer mirrors the current implementation in v5/core/*.py and v5/model.py.

↓ Scroll to explore

1. Text to Tokens

From characters to numbers

Language models do not read letters directly. They read token IDs — integers that represent subword pieces.

A BPE tokenizer splits text into the most common subword units it learned during training.

Simple mental model: text → token IDs → vectors.

2. Complex Embedding

Each token becomes a complex vector

Each token ID indexes into two embedding tables: one for the real part, one for the imaginary part.

Together they form a complex number per dimension: z = real + i · imag.

Magnitude is feature strength. Phase angle is directional identity.

"happy" and "sad" can share magnitude but have opposite phase (π apart).

3. ComplexLinear

Why no sin/cos — pure GPU matmul

Complex matrix multiplication decomposes into 4 real matrix multiplies:

Y_r = W_r·X_r − W_i·X_i
Y_i = W_r·X_i + W_i·X_r

This IS a rotation + scaling — but computed entirely with multiply-add. No trig. GPUs are built for this.

Beginner view: a real weight is a volume knob. A complex weight is volume + steering together.

4. Rotation Math (Concept Tool)

How learned rotation can be parameterized

This scene shows Cayley-style rotation math from earlier versions:

cos = (1 − a²) / (1 + a²)
sin = 2a / (1 + a²)

In current V5 code, this exact block is not part of the main path, but the intuition is useful: rotation can be learned using stable arithmetic forms.

Takeaway: rotation is a first-class operation in complex models, not just a metaphor.

5. modReLU vs V4's Broken GELU

THE critical fix

V4 applied GELU to the real part only, which breaks phase consistency. V5 applies ReLU to the magnitude, preserving phase direction.

V4: GELU on real part — phase destroyed
V5: modReLU — phase preserved
V4 pays the cost of complex numbers but destroys their benefit at every activation. modReLU fixes this.

6. Complex Gated Unit

Scale AND rotate — not just filter

SwiGLU gates produce a real scalar: they can only scale (let X% through).

The CGU gate produces a complex number: it both scales (magnitude) and rotates (phase).

"Let 70% through AND rotate by 30°" — the gate transforms what it passes.

Key idea: gates do two jobs at once — choose how much signal passes and how it is rotated.

7. Multi-Bank Fusion

Algebraic interference — for real this time

Multiple banks process the same input in parallel, each learning different useful transforms.

A content router assigns per-token weights (softmax over bank magnitudes). Learned phase rotations are applied before summing.

Aligned phases → constructive interference (amplify). Opposing phases → destructive (cancel).

Implementation note: router weights are real-valued probabilities; phase behavior comes from complex bank outputs + learned rotations.

8. Complex SSM + Parallel Scan

Memory via rotation, speed via parallelism

The state vector rotates and decays at each step: h[t] = A·h[t-1] + B·x[t].

Fast oscillation tracks local syntax. Slow oscillation tracks long-range context.

The long recurrence is evaluated with a parallel prefix scan (Blelloch-style), reducing sequential depth from O(n) to O(log n).

V4's Python loop over every timestep caused 17x slowdown. The parallel scan fixes this algorithmically.

9. Phase Attention

Sparse, complex-valued, every K layers

Score = Re(q · conj(k)) / √d — captures magnitude similarity and phase alignment in one score.

Same phase = high score. Opposite phase = negative. Orthogonal = zero.

Applied with a sliding window mask, only every K-th layer, to keep cost low.

One complex attention head captures magnitude similarity AND directional alignment in a single score.

10. Output Head

Complex → Real → Prediction

The complex representation is converted to real by concatenating real and imaginary parts.

A linear layer maps to vocabulary size, softmax gives probabilities, and the top token is the prediction.

That prediction feeds back as the next input — autoregressive generation.

Practical bridge: keep algebraic structure through the stack, convert to real only for vocabulary logits.

11. Training — The Mirror

Forward pass, loss, backward pass

Forward pass makes predictions. Cross-entropy computes error. Backprop sends gradients through all blocks to update parameters.

12. Complex Dropout

Drop features, not components

Standard dropout on real and imaginary parts independently destroys phase — it's like cutting one leg off a compass.

Complex dropout drops entire features: both real and imaginary vanish together, or both stay.

If one component of a complex feature is dropped alone, direction is corrupted. Drop both together to keep direction meaningful.
